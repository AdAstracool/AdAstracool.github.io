Hexo创建博客的步骤

发表于 2020-07-04 | 分类于 Hexo

前期准备

node.js和git的安装和环境变量配置就不说了，比较简单，教程一搜一大堆。

使用 npm install -g hexo-cli 命令安装hexo框架之后，在node.js的安装目录下建立blog文件夹，这是博客网站的根目录，然后打开cmd命令行窗口，进入blog文件夹，使用 hexo init 命令初始化博客根目录，这个命令会在将blog目录下创建一些必要的文件和文件夹。前期的准备工作就做完了。

创建新博客

在命令行窗口进入到blog目录下，使用 hexo new “新博客名” 命令就能创建一篇新博客，新的博客位于 bolg/source/_posts 目录下，是名为 新博客名.md 的文件，很多文本编辑软件都能打开，像notepad++，sublime text 3等。此时的博客里面是没有内容的，需要自己使用编辑软件写一些东西进去，这就需要知道相关的markdown语法。markdown的语法很简单，一看就明白，我在写这篇小博客的时候就是看着语法来写的。

生成静态文件

新博客在创建好之后，你可以在里面任意写想写的内容，然后使用 hexo g 命令生成博客的静态文件，我们使用Hexo框架创建的个人博客是静态的，所以需要生成.md文件对应的静态文件。生成的静态文件位于 blog/public/ 目录下，根据时间自动分成了不同的文件夹，以便于我们管理。

启动本地服务器查看效果

在生成了对应的静态文件之后，我们可以使用 hexo s 来启动我们的本地服务器，然后打开浏览器，输入 localhost:4000 （默认是4000端口）访问我们的博客，并查看效果。

部署到github.io

修改完之后，使用 npm install hexo-deployer-git –save 命令安装部署插件，安装完成之后使用 hexo d 命令就能把本地博客根目录部署到github，并使用 username.github.io 访问。

hexo clean

这条命令会删掉博客根目录下的db.json和public目录。不过hexo g编译之后这两个文件就回来了。

意外状况

在插入图片的时候，我一开始是直接截图再ctrl+v粘贴到typora里，然后保存->hexo g->hexo s->localhost:4000查看效果的时候，所有的图片都显示不出来（脑阔疼）。经过一番搜索，网上说的修改__config.yml里的 post_asset_folder: true ，然后通过 npm install hexo-asset-image –save 安装插件，我试了之后发现，修改了 post_asset_folder 之后，在 hexo -n “新博客” 的时候，的确是产生了一个.md文件和一个同名的文件夹。然后我把图片放在文件夹里，使用typora编辑.md文件时使用 这个命令插入图片，在带上文件夹名的情况下图片在typora里正常显示，但是生成的静态文件把图片和html文件放在了一起，这就导致在使用typora编辑的时候看起来显示正常的图片，在网页上会因为路径错误而显示不出来。而当我在typora里插入图片的时候不加文件夹名，即使用 插入图片时，编辑的时候typora里看不到图片，而编译后在网页上显示正常。真让人头大。在我一番努力任然没有解决这个问题的无奈下，我采用了一个笨方法：在编辑.md文件的时候，需要插入图片的地方使用两条命令，一条编辑的时候查看效果，一条网页显示用。实属无奈。
